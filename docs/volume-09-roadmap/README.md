# Volume 9: Implementation Roadmap

**Status:** Planned  
**Prerequisites:** All volumes

---

## Scope (Planned)

- Phase −1: Philosophy and principles (Volumes −1, 0.5) ← current phase
- Phase 0: Constitution ratified (Volume 0)
- Phase 1: RTN core — identity, auth, audit ledger
- Phase 2: Mission Engine core — create, plan, allocate, execute, complete
- Phase 3: RSP reference implementation — publish/subscribe, signing
- Phase 4: Operational Graph — mission graph, nodes, edges, state machines
- Phase 5: Shield MVP — first reference implementation (control room, dispatch, evidence)
- Phase 6: First participant cohort — 3–5 security companies, 1 insurer
- Phase 7: Network effects — recovery, municipality, cross-participant intelligence
- Phase 8: Resource economy — marketplace, capacity exchange
- Phase 9: Commerce integration — second reference implementation of Mission Engine
- Phase 10: Vertical expansion — Rescue, Fleet, Utility, City

## Principles for Implementation

- No code before doctrine (Phases −1 and 0 must be ratified)
- Philosophy before constitution, constitution before code
- Mission Engine before Shield (orchestration before vertical)
- RTN before Mission Engine (trust before operations)
- One city before many (Cape Town proof of concept)
- One vertical before many (Shield before Rescue)
- Integration before replacement (RSP adapters, not migration)
