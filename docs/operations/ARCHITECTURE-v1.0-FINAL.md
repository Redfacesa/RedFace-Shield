# Architecture v1.0 — Final Observation

> A platform is not validated when its code compiles.
>
> A platform is not validated when its demonstrations succeed.
>
> A platform is validated when reality changes its roadmap more often than its founders do.

**Status:** Ratified. Chapter closed.  
**Next chapter:** Operators.

## Evidence hierarchy

| Scale | What it reveals |
|-------|-----------------|
| **1 mission** | Anecdote |
| **10 missions** | Friction |
| **50 missions** | Patterns |
| **200 missions** | Product |
| **Thousands across organizations** | Platform |

Use this progression for roadmap decisions — not intuition.

---

## What mattered most

Early: what RedFace *was* — platform, security system, operating system, protocol.

Late: the **learning loop** — Deploy → Observe → Learn → Improve → Measure → Case Study → Deploy Again.

Mature products become organizations that learn, not collections of features.
